模式

vim有几种模式：

  • normal模式：可以进行光标移动与文字操作

  • insert模式：可以进行文字输入

  • visual模式：可以选中文字

  • command模式：执行vim命令

模式切换

进入时处在normal模式，切换至insert模式：

  • i：从光标前开始输入，I：从行前开始输入
  • a：从光标后开始输入，A：从行后开始输入
  • o：向下另起一行开始输入，O：向上另起一行开始输入
  • s：删除光标字符并开始输入，S：删除行并开始输入
  • c：
  • r：替换当前的字符，R：进入替换模式

切换至visual模式：

  • v直接进入visual
  • V：进入visual line模式，即行选
  • CTRL+v：进入visual block，即块选

切换至command模式 :

按ESC回退至normal模式

光标移动

在normal模式下，除了使用方向键来移动还可以使用h(←),l(→),j(↓),k(↑)

为什么vim要多此一举，可能是最初的键盘没有方向键

单词单位

  • w：移动到下一个单词开始处，W：移动到下一个大单词开始处（所谓大单词，即不忽略除空格外的分隔符）
  • e：移动到下一个单词结尾处，E：移动到下一个大单词结尾处
  • b：移动到上一个单词开始处，B：移动到上一个大单词开始处

行单位

  • 0：移动到该行的第一个字符，$：移动到一行的最后一个字符。
  • gg：移动到文件开始处，G：移动到文件结尾处

页单位

  • ctrl + f：向下移动一页
  • ctrl + b：向上移动一页
  • ctrl + d：向下移动半页
  • ctrl + u：向上移动半页
  • H：移动到屏幕开始行处，M：移动到屏幕中间行处，L：移动到屏幕结尾行处

搜索定位

  • f：字符搜索于本字符向行末搜索，F：字符搜索于本字符向行前搜索
  • t：与f类似只是定位于目标的前一个字符，T：与F类似只是定位于目标的后一个字符
  • /：向下查找字符串，？：向上查找字符串
    • n：重复执行上一个查找操作，N:重复反向执行上一个查找操作
    • CTRL-O：跳回实际的位置，CTRL-I：跳转到之前位置。
  • %： 查找最近匹配的括号
  • *：搜索当前光标处单词的其它实例

不退出normal的编辑操作

  • x：向后删除一个字符 (相当于 [del] 按键)

  • X：向前删除一个字符(相当于 [backspace] 亦即是退格键)

  • d：剪切，dd：剪切当前行

  • y：复制，yy：复制当前行

  • p：于下行开始粘贴，P：于上行开始粘贴

  • j：将光标所在行与下一行的数据结合成同一行

  • u：撤销，U：撤销行内命令

  • [Ctrl]+r：重做

VIM语句

动词(动作)：

如d，c等等

修饰语：

  • i：内部
  • a：周围
  • num：数字
  • t，T，f，F，/，?
  • 等等

名词：

  • w: 单词
  • s: 句子
  • ): 句子 (另一种操作方式)
  • p: 段落
  • }: 段落 (另一种操作方式)
  • t: 标签 ( HTML/XML)
  • b: 块 (编程语言)
  • 等等

我们可以组合==动词+ 修饰语 + 名词==来执行一个命令

VIM命令

执行外部命令：
:! 然后紧接着输入一个外部命令可以执行该外部命令。

CTRL-D：查看与输入的内容相匹配的命令

文件操作

  • :w [filename] ：另存档

  • :saveas ~/some/path/: 将文件保存到其他位置

  • :e 加载另一个文件 显示的文件是已激活的缓冲区，未显示的文件是隐藏的缓冲区

  • :ls 显示已激活的缓冲区

  • :ls! 显示所有缓冲区

  • :enew 创建一个无名的缓冲区，:w name 以name保存该文件, 然后就为这个缓冲区命了名

替换操作

替换命令

:s/old/new/g 可以替换 old 为 new。g代表整行。

输入 :#,#s/old/new/g 其中 #,# 代表的是替换操作的若干行中首尾两行的行号。
输入 :%s/old/new/g 则是替换整个文件中的每个匹配串。
输入 :%s/old/new/gc 会找到整个文件中的每个匹配串，并且对每个匹配串提示是否进行替换。

其它操作

zz把当前行设置为屏幕的中间

CTRL-G： 显示当前行的位置即文件状态

ZZ：保存并推退出

ZQ：不保存退出

CTRL-w + hjkl 在窗口之间移动

vim设置

按键映射

递归 非递归 模式
:map :noremap normal, visual, operator-pending
:nmap :nnoremap normal
:xmap :xnoremap visual
:cmap :cnoremap command-line
:omap :onoremap operator-pending
:imap :inoremap insert

所谓递归是指若a映射到b，但是b映射到c，就变成了a执行c命令

经验法则：除递归映射是必须的，否则总是使用非递归映射。

通过不给一个右值来检查你的映射。比如:nmap 显示所以普通模式下的映射，:nmap <leader> 显示所有以 <leader> 键开头的普通模式下的映射。
